## 单独使用 LLM 的问题

LLM 很强大，但有两个根本性的局限：

**1. 知识截止日期** — LLM 的训练数据有截止日期，它对那之后发生的事情一无所知。

**2. 幻觉** — 当被问到具体或冷僻的事实时，LLM 会生成听起来合理但完全错误的内容。

```python
# 问 GPT-4o 你公司内部的政策
"超过 500 美元的订单退款政策是什么？"

# 模型从未见过你的政策文件
# 它要么说不知道，要么更糟——直接编一个
```

把所有文档塞进 system prompt 也行不通。文档可能有几千页，远超任何模型的上下文窗口。

## 什么是 RAG？

**RAG（Retrieval-Augmented Generation，检索增强生成）** 是一种在查询时为 LLM 提供外部知识的技术——通过检索相关文档并将其注入 prompt，让模型基于真实内容作答。

它不再依赖模型训练时记忆的内容，而是直接把模型需要的事实喂给它。

```
不用 RAG：  用户问题 → LLM → 回答（来自训练记忆）
用 RAG：    用户问题 → 检索相关文档 → LLM + 文档 → 有依据的回答
```

## RAG 是怎么工作的？

RAG 分为两个独立的阶段：

### 阶段一：索引（提前做一次）

1. 准备你的文档（PDF、文章、数据库记录）
2. 将文档切分成更小的 chunk
3. 用向量模型将每个 chunk 转换成向量嵌入（一组捕捉语义的数字）
4. 将向量存储在向量数据库中

### 阶段二：检索 + 生成（每次查询时执行）

1. 用户提问
2. 用相同的模型将问题转换为向量
3. 在向量数据库中搜索与问题最相似的 chunk
4. 将这些 chunk 作为上下文注入 prompt
5. LLM 基于检索到的 chunk 生成有依据的回答

```python
# 简化的 RAG 流程
question = "超过 500 美元的订单退款政策是什么？"

# 第一步：对问题做向量嵌入
question_embedding = embed(question)

# 第二步：找到最相关的 chunk
relevant_chunks = vector_db.search(question_embedding, top_k=3)

# 第三步：构建带上下文的 prompt
prompt = f"""
只根据下面的上下文回答问题。

上下文：
{relevant_chunks}

问题：{question}
"""

# 第四步：生成回答
answer = llm.generate(prompt)
```

## 为什么 RAG 有效？

RAG 有效是因为你不再要求 LLM 从记忆中回忆事实，而是让它**阅读并总结**你已经检索到的段落——这是 LLM 非常擅长的任务。

模型的工作从"记住答案"变成了"从这段文字中提取答案"。

## RAG vs 微调（Fine-tuning）

一个常见的问题是：为什么用 RAG 而不是在自己的文档上微调模型？

| | RAG | 微调 |
|---|---|---|
| **知识更新** | 重新索引文档 | 重新训练模型 |
| **成本** | 低（只需索引） | 高（需要 GPU 训练） |
| **可解释性** | 可以展示来源 chunk | 不透明 |
| **适合场景** | 动态、频繁更新的数据 | 改变模型行为或风格 |

对于大多数涉及私有数据或频繁更新数据的场景，RAG 是更合适的选择。

## 面试常问

> _Q: 什么是 RAG？它解决了什么问题？_

RAG（检索增强生成）是一种在查询时用检索到的相关文档来增强 LLM 回复的技术。它解决了单独使用 LLM 的两个核心局限：知识截止日期（模型不了解训练数据截止日期之后的信息）和幻觉（模型编造它不知道的事实）。通过将检索到的上下文注入 prompt，模型可以基于真实、最新的文档来回答问题。

> _Q: RAG pipeline 的两个阶段是什么？_

索引阶段提前完成：将文档切分成 chunk，每个 chunk 转换成向量嵌入，向量存储在向量数据库中。检索与生成阶段在每次查询时执行：将用户问题做向量嵌入，在向量数据库中搜索最相似的 chunk，将这些 chunk 注入 prompt 作为上下文，LLM 生成有依据的回答。

> _Q: 为什么对于私有知识库，通常优先选择 RAG 而不是微调？_

微调把知识固化在模型的参数中，这意味着每次数据更新都需要重新训练——既昂贵又耗时。RAG 把知识保存在外部，更新数据只需重新索引文档即可。RAG 也更透明，可以向用户展示回答所依据的来源 chunk。微调更适合改变模型的行为或风格，而不是注入事实性知识。

> _Q: 在 RAG 中，"grounding"（有依据）是什么意思？_

Grounding 是指将模型的回答锚定在具体的、已检索到的来源文档上，而不是依赖训练记忆。一个有依据的回答，每一个论断都可以追溯到某个检索到的 chunk。Grounding 能减少幻觉，因为模型是在总结你提供的文字，而不是从记忆中生成事实。
