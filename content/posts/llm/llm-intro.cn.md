## 什么是大语言模型？

大语言模型（LLM，Large Language Model）是一种经过大量文本训练的 AI 模型，能够理解和生成人类语言。你给它文字，它给你文字。

LLM 是 **NLP（自然语言处理）** 的子集——NLP 是让计算机理解人类语言的更广泛领域。在 LLM 出现之前，每个 NLP 任务（翻译、摘要、文本分类）都需要单独训练一个专门的模型。LLM 改变了这一切：一个模型就能做所有事情。

ChatGPT、Claude、Gemini 都是大语言模型。它们的底层原理是一样的：**预测接下来应该出现什么文字**。

## 大语言模型是怎么工作的？

LLM 在训练阶段读取了海量文本——书籍、文章、网页、代码。训练过程中，它学习了文字之间的规律：什么词后面通常跟什么词，句子是如何组织的，概念之间如何关联。

当你发送一条消息，模型并不是在"查找"答案。它是**一个 token 一个 token 地生成回复**，每个 token 都是基于前面所有内容，选择最可能出现的下一个词。

可以把它理解成手机上的输入法联想——但它是在整个互联网上训练的，能力强得多。

## 什么是 Token？

Token 是 LLM 处理文本的最小单位。Token 不完全等于一个词，而是一段字符。

```
"Hello, world!" → ["Hello", ",", " world", "!"]  （4 个 token）
"unhappiness"   → ["un", "happiness"]             （2 个 token）
"LLM"           → ["L", "LM"]                     （2 个 token）
```

粗略估算：**1 个 token ≈ 0.75 个英文单词**，约 4 个字符。

为什么重要？因为 LLM 一次能处理的 token 数量是有上限的——这个上限叫做**上下文窗口**。

## 什么是上下文窗口？

上下文窗口是 LLM 一次请求中能"看到"的最大文本量。超出这个范围的内容，模型完全不知道。

```
上下文窗口 = 你的消息 + 对话历史 + 系统提示词
```

举个例子，如果一个模型的上下文窗口是 128,000 个 token，它大约能处理 10 万个英文单词——相当于一本小说的长度。

当对话太长超出上下文窗口，模型会开始"忘记"早期的消息，因为它们已经放不进去了。

## 为什么大语言模型会"幻觉"？

幻觉（Hallucination）是指 LLM 自信地说出不正确的内容。

这是因为 LLM 不是数据库——它不存储事实。它学习的是**文字规律**。当被问到训练数据之外的内容（或过于具体的信息）时，它会生成听起来合理的文字，即使内容是错的。

```python
# 示例：问 LLM 一个它无法可靠回答的问题
"苹果公司现在的股价是多少？"
# LLM 可能会生成一个看起来合理的数字，但完全是编造的
```

这是 **RAG**（检索增强生成）要解决的核心问题之一——通过给模型提供真实的、经过检索的事实，而不是让它单纯依赖训练数据。

## 面试常问

> _Q: 什么是大语言模型？_

大语言模型是一种在大量文本数据上训练的神经网络，能够理解和生成人类语言。它的工作原理是根据之前所有的 token，预测序列中下一个 token。常见的 LLM 包括 GPT-4o（OpenAI）、Claude（Anthropic）和 Gemini（Google）。

> _Q: 什么是 Token？_

Token 是 LLM 处理文本的基本单位。Token 不完全等于一个词，而是子词级别的块，英文中平均约 0.75 个词或 4 个字符。LLM 逐个 token 读取和生成文本，API 的计费通常也是按 token 计算的。

> _Q: 什么是上下文窗口？为什么它很重要？_

上下文窗口是 LLM 在单次请求中能处理的最大 token 数量，包含系统提示词、对话历史和当前消息。超出上下文窗口的内容会被丢弃。上下文窗口的大小在文档摘要、长对话和 RAG pipeline 等场景中至关重要，因为检索到的内容必须和查询一起放进上下文。

> _Q: LLM 中的幻觉是什么？_

幻觉是指 LLM 生成了内容错误但表现得非常自信的文字。这是因为 LLM 预测的是统计意义上最可能的文字，而不是在检索经过验证的事实。常见的解决方案包括 RAG（提供参考文档）、提示词工程（要求模型在不确定时回答"我不知道"）以及输出验证。
